AudioEngine.js (plain object):
  Signal chain PER STEM (×3: drums, bass, melody):
    Tone.Player (loop:true)
    → Tone.Volume (0 dB)
    → Tone.Filter (lowpass, 15000 Hz)
    → Tone.Reverb (decay: 2.5s, wet: 0)
    → masterVol (Tone.Volume -6 dB)
    → Destination

  Methods:
    init()                        — δημιουργεί signal chains, Tone.loaded()
    play()                        — Tone.start() + Transport + all Players
    stop()                        — stops all
    setVolume(stem, value 0-1)    — deadband 0.015, rampTo 80ms
    setStemFilter(stem, freq)     — deadband 30Hz, rampTo 100ms, range 200-15000
    setStemReverb(stem, wet 0-1)  — deadband 0.02, rampTo 100ms
    triggerLoopChange()           — drums dip -20dB for 300ms, emit audio:loopChange
    update(dt)                    — κενό (gesture events οδηγούν audio)
    destroy()                     — dispose all Tone nodes

GESTURE → AUDIO WIRING (στο main.js):

  1. gesture:fingers (LEFT hand):
     thumb-to-finger distance → PER-STEM VOLUME
     · thumb(4) ↔ index(8)  → bass volume
     · thumb(4) ↔ middle(12) → drums volume
     · thumb(4) ↔ ring(16)  → melody volume
     Normalization: norm = (dist - 0.02) / 0.18, clamped 0-1
     Δηλαδή: κλειστά δάχτυλα = σιγά, ανοιχτά = δυνατά

  2. gesture:fingers (RIGHT hand):
     thumb-to-finger distance → PER-STEM FILTER
     · thumb(4) ↔ index(8)  → bass filter freq
     · thumb(4) ↔ middle(12) → drums filter freq
     · thumb(4) ↔ ring(16)  → melody filter freq
     Logarithmic: freq = 200 * (15000/200)^norm Hz (200-15000)

  3. gesture:crossFingers:
     Left↔Right matching fingers distance → PER-STEM REVERB
     · L_index(8) ↔ R_index(8)   → bass reverb wet
     · L_middle(12) ↔ R_middle(12) → drums reverb wet
     · L_ring(16) ↔ R_ring(16)   → melody reverb wet
     Normalization: norm = (dist - 0.05) / 0.45, clamped 0-1
     Δηλαδή: χέρια κοντά = dry, χέρια μακριά = wet reverb

  Debug overlay: δείχνει per-stem V% / F% / R% κάθε ~500ms

         facingMode:'user' } })

     set a flag this.cameraAvailable = false, return without crashing.
  2. Assign stream to videoElement.srcObject, await videoElement.play()
  3. Create OffscreenCanvas(640, 480) — used to extract ImageBitmap each frame
  4. Create Worker: new Worker('./src/workers/mediapipe.worker.js')
  5. Set up worker.onmessage handler (see below)
  6. Send { type: 'INIT' } to Worker
  7. Await 'READY' response (use a Promise that resolves on type:'READY',
     rejects on type:'ERROR' or after 10 second timeout)
  8. Set this.workerBusy = false
  9. Start frame loop via requestAnimationFrame

  1. If this.workerBusy === true: skip this frame (backpressure)
  2. Draw videoElement to offscreenCanvas context
  3. const bitmap = await createImageBitmap(offscreenCanvas)
  4. this.workerBusy = true
  5. worker.postMessage({ type:'PROCESS', bitmap }, [bitmap])  // transfer ownership

  1. this.workerBusy = false
  2. For each hand (left, right):
     a. If null: updateState("hands.[side].detected", false); continue
     b. Get palm center via getPalmCenter(landmarks)
     c. Apply applySmoothing() to palmCenter x, y, z

     d. Convert to Three.js coords: mediapipeToThreeJS(cx, cy, cz)

     f. Update particles repulsion from active hand(s)
     g. Detect pinch, emit 'gesture:pinch' only if state CHANGED
     h. Update swipe history buffer (circular, 15 entries)
     i. Detect swipe, emit 'gesture:swipe' if detected
  3. emit('hands:update', { left: state.hands.left, right: state.hands.right })

  - If both hands detected: use average palm position
  - If one hand detected: use that hand's palm position

PROMPT 2.3 — GestureDetector.js

Create src/utils/GestureDetector.js as an ES Module.

Implement:

1. isPinching(landmarks):
   - landmarks: Array<{x, y, z}> (21 points, normalized 0-1)
   - Compute Euclidean distance between landmark[4] (thumb tip)
     and landmark[8] (index tip) in 2D (ignore Z for stability)
   - Returns true if distance < 0.06
   - Returns false if landmarks is null or length < 9

2. detectSwipe(historyBuffer, cooldownRef):
   - historyBuffer: Array of the last N palm X positions (landmark[0].x),
     newest at index 0
   - cooldownRef: object { lastSwipeTime: number } (passed by reference)
   - Minimum history length to evaluate: 10 entries
   - Swipe detected if abs(history[0] - history[history.length-1]) > 0.28
     within the buffer
   - Cooldown: 800ms between swipes (compare Date.now() to cooldownRef.lastSwipeTime)
   - Returns 'left', 'right', or null
   - Direction: if final X > initial X → 'right' (remember: X is mirrored)
   - On detection: update cooldownRef.lastSwipeTime = Date.now()

3. getPalmCenter(landmarks):
   - Returns the average {x, y, z} of landmarks at indices [0, 5, 9, 13, 17]
   - Returns {x:0, y:0, z:0} if landmarks is null

4. applySmoothing(current, previous, factor = 0.15):
   - Returns: previous + (current - previous) * factor
   - Works on a single number
   - factor range: 0.08 (very smooth) to 0.3 (responsive)
   - Use 0.12 as default for palm position (good balance)

Export all four functions as named exports.

PROMPT 3.2 — ParticleSystem class (add inside GraphicsEngine.js)
────────────────────────────────────────────────────────────────────

Add class ParticleSystem inside GraphicsEngine.js (not exported).

constructor(scene, quality):
  this.quality = quality
  this.frameCounter = 0
  this._initParticles(scene)

_initParticles(scene):
  COUNTS = { low: 3000, medium: 8000, high: 15000 }
  const count = COUNTS[this.quality]

  // Home positions (Float32Array, stays constant)
  this.homePositions = new Float32Array(count * 3)
  // Current positions (updated each frame)
  this.positions = new Float32Array(count * 3)
  // Velocities
  this.velocities = new Float32Array(count * 3)

  for each particle i:
    homeX = random(-8, 8), homeY = random(-6, 6), homeZ = random(-5, 2)
    homePositions[i*3..] = homeX, homeY, homeZ
    positions[i*3..] = homeX, homeY, homeZ
    velocities[i*3..] = 0, 0, 0

  // Color: gradient blue→cyan with variation
  colors = new Float32Array(count * 3)
  for each particle: mix 0x0033ff and 0x00ffff with random blend

  geometry = new THREE.BufferGeometry()
  geometry.setAttribute('position', new THREE.BufferAttribute(positions, 3))
  geometry.setAttribute('color', new THREE.BufferAttribute(colors, 3))

  material = new THREE.PointsMaterial({
    size: 0.02,
    vertexColors: true,
    transparent: true,
    blending: THREE.AdditiveBlending,
    depthWrite: false,
    sizeAttenuation: true
  })

  this.points = new THREE.Points(geometry, material)
  scene.add(this.points)

update(deltaTime, particleState):
  this.frameCounter++
  // In LOW quality: skip every other frame
  if (this.quality === 'low' && this.frameCounter % 2 !== 0) return

  const { repulsionX, repulsionY, repulsionStrength } = particleState
  const REPULSION_RADIUS = 1.5
  const SPRING_STRENGTH = 0.02
  const DAMPING = 0.95
  const MAX_FORCE = 0.05

  for each particle i:
    px = positions[i*3], py = positions[i*3+1], pz = positions[i*3+2]
    hx = homePositions[i*3], hy = homePositions[i*3+1], hz = homePositions[i*3+2]

    // Repulsion from hand
    dx = px - repulsionX, dy = py - repulsionY
    dist = sqrt(dx*dx + dy*dy)
    if (dist < REPULSION_RADIUS && repulsionStrength > 0 && dist > 0.001):
      force = min(repulsionStrength / (dist * dist), MAX_FORCE)
      velocities[i*3]   += (dx / dist) * force
      velocities[i*3+1] += (dy / dist) * force

    // Spring back to home
    velocities[i*3]   += (hx - px) * SPRING_STRENGTH
    velocities[i*3+1] += (hy - py) * SPRING_STRENGTH
    velocities[i*3+2] += (hz - pz) * SPRING_STRENGTH

    // Damping
    velocities[i*3]   *= DAMPING
    velocities[i*3+1] *= DAMPING
    velocities[i*3+2] *= DAMPING

    // Update position
    positions[i*3]   += velocities[i*3]
    positions[i*3+1] += velocities[i*3+1]
    positions[i*3+2] += velocities[i*3+2]

  geometry.attributes.position.needsUpdate = true

setQuality(level): rebuild particles with new count if level changed

    this.spheres.forEach(s => s.visible = false)
    this.lines.forEach(({ line }) => line.visible = false)
    return

  this.pulsePhase += 0.05
  const pulseScale = 1 + Math.sin(this.pulsePhase) * 0.05
  const pinching = handData.pinching  // already set in state by TrackingEngine

  // Update landmark positions
  handData.landmarks.forEach((lm, i) => {
    const { x, y, z } = mediapipeToThreeJS(lm.x, lm.y, lm.z)
    this.landmarkPositions[i].set(x, y, z)

    this.spheres[i].position.copy(this.landmarkPositions[i])
    this.spheres[i].scale.setScalar(pulseScale)
    this.spheres[i].visible = true

    // Pinch highlight on thumb tip (4) and index tip (8)
    const isPinchPoint = (i === 4 || i === 8)
    this.spheres[i].material.color.setHex(
      (isPinchPoint && pinching) ? 0xff00ff : 0x00ffff
    )
  })

  // Update connection lines
  this.lines.forEach(({ line, a, b }) => {
    const positions = line.geometry.attributes.position
    positions.setXYZ(0, ...this.landmarkPositions[a].toArray())
    positions.setXYZ(1, ...this.landmarkPositions[b].toArray())
    positions.needsUpdate = true
    line.visible = true
  })

class AudioEngine:

async init():
  Tone.Transport.bpm.value = 120

  // Signal chain setup (done BEFORE loading samples)
  this.masterVol = new Tone.Volume(-6).toDestination()

  // Drums: direct to master
  this.drumsVol = new Tone.Volume(0).connect(this.masterVol)
  this.drums = new Tone.Player({
    url: "assets/audio/drums.mp3",
    loop: true,
    autostart: false
  }).connect(this.drumsVol)

  // Bass: Volume → Filter → master
  this.bassFilter = new Tone.Filter(8000, "lowpass").connect(this.masterVol)
  this.bassVol = new Tone.Volume(-6).connect(this.bassFilter)
  this.bass = new Tone.Player({
    url: "assets/audio/bass.mp3",
    loop: true,
    autostart: false
  }).connect(this.bassVol)

  // Melody: Volume → Filter → master
  this.melodyFilter = new Tone.Filter(8000, "highpass").connect(this.masterVol)
  this.melodyVol = new Tone.Volume(-6).connect(this.melodyFilter)
  this.melody = new Tone.Player({
    url: "assets/audio/melody.mp3",
    loop: true,
    autostart: false
  }).connect(this.melodyVol)

  await Tone.loaded()
  this.samplesLoaded = true
  console.log("AudioEngine: all samples loaded")

async play():
  if (!this.samplesLoaded) return
  await Tone.start()         // REQUIRED: browser autoplay policy
  Tone.Transport.start()
  this.drums.start(0)
  this.bass.start(0)
  this.melody.start(0)
  updateState("isPlaying", true)
  emit('audio:play')

stop():
  Tone.Transport.stop()
  this.drums.stop()
  this.bass.stop()
  this.melody.stop()
  updateState("isPlaying", false)
  emit('audio:stop')

setVolume(stem, value):  // value: 0.0 to 1.0
  const db = value < 0.001 ? -Infinity : Tone.gainToDb(value)
  const node = { drums: this.drumsVol, bass: this.bassVol, melody: this.melodyVol }[stem]
  if (node) node.volume.rampTo(db, 0.05)
  updateState(`audio.${stem}Volume`, value)
  emit('audio:volumeChange', { stem, value })

setFilter(type, frequency):
  const clamped = Math.max(200, Math.min(15000, frequency))
  const filter = type === 'lowpass' ? this.bassFilter : this.melodyFilter
  filter.frequency.rampTo(clamped, 0.1)
  updateState("audio.filterCutoff", clamped)

triggerLoopChange():
  // Schedule at next bar boundary (4 beats at 120bpm = 2 seconds)
  const nextBar = Tone.Transport.nextSubdivision("1m")
  Tone.Transport.scheduleOnce(() => {
    // Brief drums dip — swaps feel of the loop
    this.drumsVol.volume.rampTo(-20, 0.1)
    Tone.Transport.scheduleOnce(() => {
      this.drumsVol.volume.rampTo(0, 0.3)
    }, "+0.3")
    emit('audio:loopChange')
  }, nextBar)

update(deltaTime):
  if (!state.isPlaying) return
  // Left hand Y → bass volume (inverted: hand up = loud)
  if (state.hands.left.detected):
    const bassVol = 1 - state.hands.left.y   // y is already mapped to 0-1 range
    // Clamp to 0-1
    this.setVolume('bass', Math.max(0, Math.min(1, bassVol)))
  // Right hand Y → melody volume
  if (state.hands.right.detected):
    const melVol = 1 - state.hands.right.y
    this.setVolume('melody', Math.max(0, Math.min(1, melVol)))

  IMPORTANT: y in StateManager is in Three.js space (-3.3 to +3.3).
  Normalize before using: normY = (y + 3.3) / 6.6
  Then: volume = 1 - normY

destroy(): Tone.Transport.stop(), dispose all nodes

PLANET_DATA = [
  { name:'earth',   radius:0.4, distance:4,   orbitSpeed:0.005,  texture:'earth.jpg'   },
  { name:'mars',    radius:0.3, distance:6,   orbitSpeed:0.003,  texture:'mars.jpg'    },
  { name:'jupiter', radius:0.7, distance:9,   orbitSpeed:0.001,  texture:'jupiter.jpg' },
  { name:'saturn',  radius:0.6, distance:12,  orbitSpeed:0.0007, texture:null          }
]

CAMERA_TARGETS sequence: ['sun', 'earth', 'mars', 'jupiter', 'saturn']
  (cycles on each 'audio:loopChange' event)

constructor(scene, camera, quality):
  Create Sun:
    - SphereGeometry(1.5, 32, 32)
    - MeshBasicMaterial, texture only if quality !== 'low' (else color 0xffaa44)
    - Add a slightly larger sphere (SphereGeometry(1.8, 16, 16)) as glow:
        MeshBasicMaterial, color 0xff6600, transparent, opacity 0.15,
        AdditiveBlending, depthWrite false
    - Slow Y-axis rotation: 0.001 rad/frame

  Create Planets (array):
    For each planet:
      - SphereGeometry, MeshStandardMaterial
      - If quality !== 'low' and texture exists: load texture
      - Else: fallback colors (earth=0x2255aa, mars=0xcc3300, jupiter=0xddaa55, saturn=0xddcc88)
      - this.orbitAngles[i] = random(0, TWO_PI)  (stagger initial positions)

  this.targetIndex = 0
  this.cameraLerpT = 1.0  (1.0 = animation complete)
  this.cameraLerpFrom = new THREE.Vector3()
  this.cameraLerpTo = new THREE.Vector3()
  this.lookAtLerpTo = new THREE.Vector3()

  on('audio:loopChange', () => this._advanceTarget())

_advanceTarget():
  this.targetIndex = (this.targetIndex + 1) % CAMERA_TARGETS.length
  const targetName = CAMERA_TARGETS[this.targetIndex]
  updateState('camera.currentTarget', targetName)
  updateState('camera.isAnimating', true)

  // Determine camera destination
  const targetPos = targetName === 'sun'
    ? new THREE.Vector3(0, 1.5, 4)  // offset from sun center
    : this._getPlanetWorldPos(targetName).clone().add(new THREE.Vector3(0, 0.5, 1.5))

  this.cameraLerpFrom.copy(this.camera.position)
  this.cameraLerpTo.copy(targetPos)
  this.lookAtLerpTo.copy(targetName === 'sun' ? new THREE.Vector3() : this._getPlanetWorldPos(targetName))
  this.cameraLerpT = 0.0

  // Trigger color palette transition
  this.colorSystem.transitionTo(targetName)

update(deltaTime):
  // Rotate sun
  this.sun.rotation.y += 0.001

  // Orbit planets
  this.planets.forEach((planet, i) => {
    this.orbitAngles[i] += PLANET_DATA[i].orbitSpeed
    const d = PLANET_DATA[i].distance
    planet.position.set(
      Math.cos(this.orbitAngles[i]) * d,
      0,
      Math.sin(this.orbitAngles[i]) * d
    )
    planet.rotation.y += 0.005
  })

  // Smooth camera animation (lerp over ~2 seconds)
  if (this.cameraLerpT < 1.0) {
    this.cameraLerpT = Math.min(1.0, this.cameraLerpT + deltaTime * 0.5)
    const t = this._easeInOut(this.cameraLerpT)
    this.camera.position.lerpVectors(this.cameraLerpFrom, this.cameraLerpTo, t)
    this.camera.lookAt(this.lookAtLerpTo)
    if (this.cameraLerpT >= 1.0) updateState('camera.isAnimating', false)
  }

_easeInOut(t): return t < 0.5 ? 2*t*t : -1+(4-2*t)*t

PROMPT 7.1 — Keystone Calibration (Projection Mapping)
────────────────────────────────────────────────────────────────────

Add keystone correction to GraphicsEngine.js using a post-process shader pass.

Technical approach:
  1. Render Three.js scene to WebGLRenderTarget (offscreen)
  2. Apply quad-warp shader (perspective-correct)
  3. Render warped result to final canvas

Calibration mode toggle: Ctrl+Shift+K

When calibration mode ON:
  - Overlay 4 draggable HTML corner handles on canvas
  - Initial positions: corners of canvas (TL, TR, BL, BR)
  - User drags handles to match projector trapezoid
  - Save 4 corner positions to localStorage key 'dj-viz-keystone'
  - Load on startup if exists

Warp shader (WebGL fragment shader):
  - Uniforms: sampler2D uTexture, vec2 uTL, uTR, uBL, uBR (corner positions 0-1)
  - For each fragment: compute bilinear UV from warped quad
  - Use perspective-correct interpolation (homogeneous coords)

Performance:
  - When calibration mode OFF: render directly (NO extra render pass)
  - When calibration mode ON: add one extra fullscreen quad render pass

ΠΡΟΒΛΗΜΑ: "Tone.js δεν παίζει ήχο"
  ΛΥΣΗ: Browser απαιτεί user gesture πριν από audio.
  Πάντα κάλεσε Tone.start() μέσα σε click/keypress handler (το Spacebar shortcut
  στο main.js το κάνει αυτό σωστά).

ΠΡΟΒΛΗΜΑ: "MediaPipe Worker δεν φορτώνει — SecurityError / NetworkError"
  ΛΥΣΗ 1: Τρέχεις από file://? Άνοιξε με Live Server (http://)
  ΛΥΣΗ 2: CORS: το MediaPipe CDN επιτρέπει cross-origin. Αν χρησιμοποιείς
           τοπικό αρχείο, σέρβαρέ το μέσω Live Server.
  ΛΥΣΗ 3: Αν το importScripts() αποτυγχάνει: δοκίμασε να κατεβάσεις το
           vision_bundle.js τοπικά στο src/workers/vision_bundle.js
           και χρησιμοποίησε: importScripts('/src/workers/vision_bundle.js')

ΠΡΟΒΛΗΜΑ: "MediaPipe error: Cannot use GPU delegate inside Worker"
  ΛΥΣΗ: Άλλαξε delegate σε 'CPU' στο mediapipe.worker.js
  Η απόδοση είναι ελαφρά χαμηλότερη αλλά αρκετή για Lite model.

ΠΡΟΒΛΗΜΑ: "Χέρι: jitter / τρεμοπαίζει"
  ΛΥΣΗ 1: Μείωσε factor στο applySmoothing() από 0.12 σε 0.07
  ΛΥΣΗ 2: Αύξησε swipe history buffer από 15 σε 20 frames
  ΛΥΣΗ 3: Απόρριψε landmarks με confidence < 0.6 (πρόσθεσε threshold)

ΠΡΟΒΛΗΜΑ: "FPS κάτω από 20"
  ΛΥΣΗ 1: Πάτα Q (LOW quality) — άμεση βελτίωση
  ΛΥΣΗ 2: Μείωσε webcam resolution: width:320, height:240 στο TrackingEngine
  ΛΥΣΗ 3: Frame skip: process 1 frame στα 2 από webcam
  ΛΥΣΗ 4: Αν Chrome: ελέγξε chrome://gpu — WebGL πρέπει να έχει "Hardware accelerated"

ΠΡΟΒΛΗΜΑ: "Coordinates χεριού ανεστραμμένα / δεξί-αριστερά"
  ΛΥΣΗ: Στο CoordinateMapper, η X πρέπει να ανακλάται:
  threeX = -(nx * 8.8 - 4.4)
  Το αρνητικό πρόσημο κάνει mirror. Αν παραλείφθεί, το hand tracking είναι
  σαν να βλέπεις σε mirror — κάθε κίνηση δεξιά στον κόσμο πηγαίνει αριστερά.

ΠΡΟΒΛΗΜΑ: "Ήχος out of sync — κολλάει κάθε φορά που ενημερώνεται volume"
  ΛΥΣΗ: ΜΗΝ καλείς setVolume() κάθε frame.
  Χρησιμοποίησε rampTo() με 0.05s εντός του setVolume() (ήδη στον κώδικά σου).
  Αν εξακολουθεί: αύξησε ramp time σε 0.1s.

ΠΡΟΒΛΗΜΑ: "Three.js canvas δεν καλύπτει όλη την οθόνη / λευκό margin"
  ΛΥΣΗ: Βεβαιώσου ότι στο CSS:
    body { margin: 0; overflow: hidden; }
    #main-canvas { display: block; width: 100vw; height: 100vh; }

ΠΡΟΒΛΗΜΑ: "importmap δεν δουλεύει (TypeError: Failed to resolve module)"
  ΛΥΣΗ: importmap υποστηρίζεται μόνο σε Chrome 89+, Firefox 108+, Safari 16.4+
  Βεβαιώσου ότι το browser είναι ενημερωμένο.
  Fallback: αντικατάστησε import 'three' με full CDN URL στο importmap.

ΠΡΟΒΛΗΜΑ: "Το Worker δεν λαμβάνει το ImageBitmap (error: could not be cloned)"
  ΛΥΣΗ: Βεβαιώσου ότι περνάς το bitmap ως Transferable:
  worker.postMessage({ type:'PROCESS', bitmap }, [bitmap])
  Το δεύτερο argument [bitmap] είναι το transferList. Χωρίς αυτό, προσπαθεί
  να κάνει structured clone που αποτυγχάνει για ImageBitmap.
